# 县级医院陪诊管理系统-项目文档

更新时间：2026-03-04

## 1. 项目概况
- 项目由 `backend`（Spring Boot + MyBatis-Plus + MySQL）和 `frontend`（Vue3 + Pinia + Element Plus + Vite）组成。
- 目标是支持患者下单、陪诊员接单服务、管理端运营与审核。
- 当前采用前后端分离开发模式，同时后端内置一套静态管理后台页面（`http://localhost:8080`）。

## 2. 当前已实现功能模块

## 2.1 登录与账号体系
- 双端独立登录页：
- 患者登录：`/patient/login`
- 陪诊员登录：`/companion/login`
- 根路径 `/` 默认跳转登录页（患者登录页）。
- 支持同一浏览器下患者端、陪诊员端分别保持登录态（角色会话隔离存储）。
- 注册分流：
- 患者注册：`POST /api/auth/register/patient`
- 陪诊员注册：`POST /api/auth/register/companion`
- 兼容旧接口：`POST /api/auth/register`（按患者处理）。
- 陪诊员审核状态控制：
- `status=2` 待审核（不可登录）
- `status=1` 已通过（可登录）
- `status=0` 已驳回/禁用（不可登录）

## 2.2 用户资料与用户名
- 用户表新增 `username` 字段并已接入业务。
- 注册页新增用户名输入（必填，最长 20）。
- 登录返回用户名并在前端按角色缓存。
- 顶部栏右上角显示用户名，不再显示手机号。
- 个人中心支持修改用户名：
- 接口：`PUT /api/auth/profile/username`
- 患者端个人中心可修改
- 陪诊员端个人中心可修改

## 2.3 患者端功能
- 首页（医院选择）：`/patient/home`
- 下单页：`/patient/orders/create`
- 订单列表：`/patient/orders`
- 订单详情：`/patient/orders/:orderNo`
- 评价页：`/patient/review`
- 个人中心：`/patient/profile`
- 下单后可在详情页进行“立即支付”。
- 订单查询隔离：患者端仅查询自己的订单（按 `patientPhone` 过滤）。

## 2.4 陪诊员端功能
- 接单大厅：`/companion/hall`
- 服务记录：`/companion/records`
- 收入统计：`/companion/income`
- 提现页：`/companion/withdraw`
- 个人中心：`/companion/profile`
- 接单大厅支持自动轮询刷新（用于及时看到新支付订单）。

## 2.5 订单核心流程
- 创建订单：`POST /api/orders`
- 支付订单：`PUT /api/orders/{orderNo}/pay`
- 状态推进：`PUT /api/orders/{orderNo}/next`
- 取消订单：`PUT /api/orders/{orderNo}/cancel`
- 状态主流程：
- `UNPAID -> WAITING_ACCEPT -> ACCEPTED -> IN_SERVICE -> TO_CONFIRM -> COMPLETED`

## 2.6 管理后台（后端静态页面）
- 地址：`http://localhost:8080`
- 管理员固定账号：`root`
- 管理员固定密码：`123456`
- 已实现菜单：
- 运营概览
- 订单管理
- 新建订单
- 医院管理
- 陪诊员审核
- 陪诊员审核接口：
- 列表：`GET /api/admin/companions`
- 通过：`PUT /api/admin/companions/{id}/approve`
- 驳回：`PUT /api/admin/companions/{id}/reject`

## 2.7 数据库与脚本
- 初始化脚本：`sql/init.sql`
- 已包含 `user.username` 字段与示例数据。
- 历史迁移脚本：
- `sql/migrate_user_unique_phone_role.sql`（手机号+角色唯一）
- `sql/migrate_user_add_username.sql`（新增用户名并补默认值）
- 启动脚本：
- `backend/start-backend.ps1`
- `backend/start-backend.cmd`

## 3. 主要接口清单（当前）
- 认证：
- `POST /api/auth/register`
- `POST /api/auth/register/patient`
- `POST /api/auth/register/companion`
- `POST /api/auth/login`
- `PUT /api/auth/profile/username`
- 订单：
- `GET /api/orders`（支持 `patientPhone` 可选过滤）
- `GET /api/orders/{orderNo}`
- `POST /api/orders`
- `PUT /api/orders/{orderNo}/pay`
- `PUT /api/orders/{orderNo}/next`
- `PUT /api/orders/{orderNo}/cancel`
- 医院：
- `GET /api/hospitals`
- 管理-陪诊审核：
- `GET /api/admin/companions`
- `PUT /api/admin/companions/{id}/approve`
- `PUT /api/admin/companions/{id}/reject`

## 5. 运行说明（简版）
- 后端：
- 进入 `backend` 目录，执行 `start-backend.ps1` 或 `start-backend.cmd`
- 前端：
- 进入 `frontend` 目录，执行 `npm run dev`
- 访问：
- 前端：`http://localhost:5173`
- 后端管理页：`http://localhost:8080`
